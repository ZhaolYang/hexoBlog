日常工作整理的一些实用工具推荐

发表于 2017-05-20 | 分类于 实用小工具

作为一名前端搬砖员，手上必须要有一点装b利器，接下来我就推荐一些日常工作中个人常用的好用的小工具

1.tinypng.com

一个很棒的图片压缩网站，在日常的前端页面开发工作中，难免会需要很多切图，特别是当一个页面的切图非常多的时候，为了进一步减少图片资源所占空间，优化页面载入速度，可以使用tinypng,能够在保证图片质量不下降的情况下，最大限度的压缩图片大小。压缩率贼高！还免费！前端优化必备神器有木有？！

使用方法：
1.打开网站

2.把需要压缩的图片选中拖入网站中即可，一次最多拖入20张图片，每张图片不能超过5M大小限制，但是绝对能够应付绝大多数场景，如有大图压缩需求，可以选择付费pro版~

3.压缩完毕，点击下载按钮，下载下来，完成

2.caniuse

干前端这行的，最怕的问题之一就是浏览器兼容问题了，有时候新学了个巨帅的css3属性，想要在项目中使用起来装个b什么的，却往往被万恶的ie兼容需求给击败。没办法，我们只能转向用兼容性较好的的方法来写页面，但是那么多css、js属性的兼容性我们怎么能全部记住呢，总不能每次用到都要去查一下百度吧，这里推荐一个网站，caniuse，顾名思义，这个网站就是用来查询某个元素我能不能用。

使用方法：
1.打开网站

2.输入你要查询的属性


3.需要注意的是，一般查询结果默认都不会完全显示的，这里我们需要点击 show all 按钮查看完整的兼容性内容，这时就能看到我们查询的属性兼容到ie8，美滋滋~

3.browserhacks

珍爱生命，远离ie678，出来搬砖，难免会踩坑，特别是那些需要做低版本浏览器兼容的网站，真的是做的蛋疼。。直到我遇见了它。。browserhacks，查询各种低版本浏览器hack写法，涵盖选择器hack，js hack, css属性hack,media hack，连恶心的ie独有的条件语句都有啊有木有！

2.输入你要查询的浏览器版本，以ie7为例

简单描述一下，左边是你的vps，右边是你的pc，而中间的墙就是阻挡你探索国外信息的墙咯，本教程就是使用ShadowSocks（影梭）工具来连接你的pc和你的vps，你发出的所有网络访问通过你的梯子（即shadowsocks，下文全部使用简写ss）发送给你的vps，这些网络请求由你的vps来代劳发起，由于vps是在海外的，访问海外网站自然是无压力啦，然后获取的所有网络资源再通过ss返回给你，这样就实现了通过vps搭建ss来翻墙啦，怎么样，是不是听起来很高大上？接下来就一步步教你如何实现！

从input标签开始介绍

对form表单来说，input标签是最重要也是最常用的form表单元素，该标签有多个type类型，每种类型又对应着不同的功能，首先来举几个例子。

  • type = “text”,文本类输入框，一般用于输入账号等明文信息

  • type = “password”，密码输入框，以暗文展示。

  • type = “radio”，单选框，一般用于选择性别等单选信息，name值相同的radio标签是一组，且只能单选，value值及该标签的值

  • type = “checkbox”，多选框，和单选框类似，唯一区别是可以多选。

  • textarea标签，该标签为独立标签，为文本域，用来输入多行文本，此处要注意的是该标签不是自闭和标签。

  • select标签，下拉选择框，其中每个option对呀下拉框的值

  • 提交，当我们填写完表单信息后，需要将填写完成的表单内容提交到服务器，此时需要一个提交按钮

  • 但是，当我们仅仅只是需要一个按钮而不需要表单进行提交操作时，该怎么办呢？小菜一碟！此时只需要把type值设为button即可，这样我们就得到一个不具备提交功能的按钮啦。

最后，再回到form

当我们点击提交按钮之后，表单内容会被浏览器通过我们设置的提交方式来提交给我们指定的地址，那么提交方式和地址从哪里来的呢？我们在回头看看示例的form标签。form标签的action属性指定我们的表单数据提交的地址，method指定提交方式。

form表单的提交方式有两种，get和post。这两种方式有什么区别呢？
1.本质

Get是向服务器发索取数据的一种请求，而Post是向服务器提交数据的一种请求

2.服务器端获取值的方法

get方式提交的数据，服务器端使用request.QueryString获取变量的值

post方式提交的数据，服务器端使用request.Form获取数据

3.安全性

get方式安全性低，post方式较安全。但是post方式执行效率要比get方式差一些。

4.机制

get是把参数数据队列加到提交表单的action属性所指的URL中，如：http://www.xxx.com?sessonid=db23434&name=hongten&age=20。 在URl中，值和表单南日各个字段一一对应，并且这些在URl中对用户来说是可见的，即用户时可以看到的。如：name=hongten。

post是通过HTTP post机制，将表单内各个字段与其内容放置在HTML HEADER内一起传送到action属性所指的URL地址，对于用户来说，这是透明的。

5.大小

URL不存在参数上限的问题，HTTP协议规范没有对URL长度进行限制。这个限制是特定的浏览器及服务器对它的限制。IE对URL长度的限制是2083字节(2K+35)。对于其他浏览器，如Netscape、FireFox等，理论上没有长度限制，其限制取决于操作系统的支持。
